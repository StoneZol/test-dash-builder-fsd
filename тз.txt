Тестовое задание — Dashboard Builder
Цель
Необходимо разработать приложение-конструктор дашбордов. Пользователь может собирать
собственный дашборд из готовых виджетов, менять их расположение и сохранять
конфигурацию.
Основной упор задания — качество архитектуры проекта, а не количество реализованных
функций.
Функциональные требования
Реализовать:
отображение списка доступных виджетов
добавление виджетов на дашборд
удаление виджетов
изменение порядка виджетов (Drag & Drop)
сохранение конфигурации пользователя
восстановление конфигурации после перезагрузки страницы
Минимальный набор виджетов:
Таблица
Статистика
График
Новостная карточка
Данные получать через любой публичный апи например https://restcountries.com или
https://github.com/r-spacex/SpaceX-API
Тут на ваш выбор
Технические требования
Обязательно использовать:
React 18
TypeScript
Next.js App Router
React Query
Zustand
SCSS Modules
FSD Architecture
Ai
На что мы обращаем основное внимание
При проверке задания для нас важнее всего:
организация проекта по FSD
грамотное разделение ответственности между слоями
использование React Query для работы с серверными данными
минимизация лишнего состояния в Zustand
корректная работа SSR там, где это имеет смысл
Идеальный кандидат пишет код так, чтобы его можно было без проблем поддерживать в
команде, поэтому также ожидаем документацию
Работа с API
Все серверные данные должны получаться через React Query.
Желательно использовать:
query keys
кеширование
обработку loading/error состояний
инвалидацию после изменений данных
Архитектура
Проект должен быть организован по Feature-Sliced Design.
Окружения
Необходимо настроить два окружения:
Development
Production
Они должны отличаться не только названием.
Например:
разные API endpoints
различающийся баннер в интерфейсе
разные значения feature flags
разные настройки логирования
разные переменные окружения
Важно показать понимание работы с environment-переменными в Next.js.
Деплой
Проект необходимо развернуть в двух окружениях:
Development
Production
Использование GitHub Pages не допускается, так как оно не подходит для Next.js SSRприложений.
Можно использовать любой удобный вариант:
Vercel
Docker + VPS
Railway
Render
другой сервис, поддерживающий Next.js
В результате должны быть доступны две отдельные ссылки.

подготовленную с помощью AI и при необходимости дополненную вручную.
Она должна содержать:
описание архитектуры проекта
описание выбранных решений
структуру FSD

используемые технологии
краткое описание компромиссов и того, что можно было бы улучшить при наличии
дополнительного времени
